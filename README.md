. Business Problem
Organizations need a centralized system to track internal service requests with workflow tracking, audit logging, and reporting capabilities.
________________________________________
2. Functional Requirements
•	Users can create service requests
•	Requests can be assigned to other users
•	Requests move through workflow stages
•	Comments can be added
•	Attachments supported
•	Audit log captures changes
•	Role-based access control
________________________________________
3. Non-Functional Requirements
•	RESTful API
•	JWT Authentication
•	SQL Server backend
•	Clean layered architecture
•	Optimized queries
•	Unit tested business rules
